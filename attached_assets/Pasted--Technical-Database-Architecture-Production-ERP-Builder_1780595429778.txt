# Technical Database Architecture

## Production ERP Builder

### Цель документа

Данный документ описывает архитектуру хранения данных платформы.

Платформа является Metadata Driven системой.

Это означает, что:

* страницы создаются через интерфейс;
* сущности создаются через интерфейс;
* поля создаются через интерфейс;
* роли создаются через интерфейс;
* workflow создаются через интерфейс;

Без изменения кода.

Поэтому база данных должна хранить не только бизнес-данные, но и саму структуру платформы.

---

# Общая архитектура

База данных должна состоять из двух уровней.

## Уровень 1 — Core Metadata

Описывает структуру платформы.

Например:

* страницы
* сущности
* поля
* роли
* права
* workflow

---

## Уровень 2 — Business Data

Хранит пользовательские данные.

Например:

* проекты
* азманот
* изделия
* счета

# Обязательное требование

При разработке любой новой функции разработчик обязан сначала проверить:

Можно ли реализовать функцию через существующую metadata architecture?

Если ответ "да", реализация через hardcoded код запрещена.

Любая новая бизнес-логика должна конфигурироваться через данные в PostgreSQL, а не через изменение исходного кода.

Это является основным архитектурным принципом платформы.
