# Production ERP Configuration V1

## Цель настройки

Настроить платформу таким образом, чтобы сотрудники могли работать максимально похоже на текущую систему Google Sheets.

Приоритет:

1. Минимальные изменения привычного процесса работы.
2. Максимально быстрый запуск.
3. Использование существующего функционала платформы.
4. Не создавать новый функционал.
5. Не изменять архитектуру платформы.

---

# Основная концепция V1

Несмотря на существование логических сущностей:

* Проект
* Азмана
* Изделие

основной рабочей сущностью системы является:

## Items (Изделия)

Каждая строка соответствует одному изделию.

Именно так сейчас работают сотрудники в Google Sheets.

---

# Главная рабочая страница

Создать страницу:

```text
Управление проектами
```

Тип:

Обычная страница сущности.

---

Создать сущность:

```text
Items
```

Эта сущность является центральной сущностью всей системы.

---

# Структура работы

Пользователь не создает отдельно:

* проект
* азману
* изделие

Пользователь создает сразу изделие.

При создании строки он заполняет:

```text
Проект
Азмана
Изделие
```

как это происходит сейчас в Google Sheets.

---

---

# Логика создания записи

Менеджер открывает страницу:

```text
Управление проектами
```

Добавляет новую строку.

Заполняет:

Проект

Азмана

Изделие

и остальные поля.

Система должна позволять работать максимально похоже на Google Sheets.

Редактирование должно происходить непосредственно в ячейках таблицы.

---

# Производители

Не создавать отдельные сущности производителей для работы.

Использовать поле:

```text
manufacturer
```

Тип:

Select

Значения:

Создаются администратором.

Например:

```text
Hamada
Suleiman
...
```

---

# Представление Hamada

Создать View:

```text
Hamada
```

Фильтр:

```text
manufacturer = Hamada
```

---

Показывать только поля необходимые производителю.

Скрыть:

```text
client_unit_price
client_total_price
designer_cost
profit
financial_fields
```

---

Производитель должен видеть только свои изделия.

---

# Страница Логистика

Создать страницу:

```text
Логистика
```

Использовать ту же сущность:

```text
Items
```

Не создавать новую сущность.

---

Создать отдельное представление.

Показывать:

Только поля связанные с:

```text
производством
покраской
доставкой
монтажом
```

---

Финансовые поля скрыть.

# Страница Производство

Создать страницу:

```text
Производство
```

Использовать сущность:

```text
Items
```

---

Создать представления по производителям.

Например:

```text
Hamada
Suleiman
```

---

# Роли

## Администратор

Полный доступ.

---

## Управляющий проектами

Видит все записи.

Редактирует все записи.

Видит все поля.

---

## Менеджер

Видит все записи.

Редактирует свои записи.

Не может удалять записи.

---

## Производитель

Видит только записи:

```text
manufacturer = current manufacturer
```

Видит только разрешенные колонки.

---

## Финансовый директор

Видит все записи.

Видит все финансовые поля.

---

# Workflow V1

На первом этапе использовать минимальное количество статусов.

Создать:

```text
Новая запись

Передано производителю

В производстве

Готово

Передано в логистику

Доставлено

Смонтировано

Закрыто
```

---

Не усложнять workflow.

Главная цель:

Повторить существующий процесс.

---

# Архивирование

Статус:

```text
Закрыто
```

является архивным.

Через:

```text
30 дней
```

после закрытия запись автоматически архивируется.

---

# Важное требование

На первом этапе категорически запрещено:

* создавать новые бизнес-процессы;
* придумывать новые сущности;
* усложнять workflow;
* менять существующую логику работы сотрудников.

Цель V1:

Максимально точно воспроизвести текущую работу Google Sheets внутри существующей платформы.

После запуска и тестирования допускается переход к более нормализованной модели данных через отдельные сущности Projects, Orders и Items.
