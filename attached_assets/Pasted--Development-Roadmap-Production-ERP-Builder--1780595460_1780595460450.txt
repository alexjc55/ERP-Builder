# Development Roadmap

## Production ERP Builder

### Цель документа

Этот документ определяет порядок разработки платформы.

Важно:

Запрещено пытаться реализовать всю систему одновременно.

Каждый этап должен быть полностью завершен, протестирован и только после этого можно переходить к следующему этапу.

---

# Общий принцип разработки

Платформа должна строиться снизу вверх.

Сначала создается фундамент.

Затем конструктор.

Затем права доступа.

Затем бизнес-процессы.

Затем интеграции.

---

## Core Infrastructure

Цель:

Создать техническое ядро платформы.

---

Необходимо реализовать:

PostgreSQL

Backend API

Frontend React

Authentication Layer

Authorization Layer

API Layer

Metadata Layer

Audit Layer

---

Результат этапа:

Пустая система.

Можно войти в систему.

Можно создавать пользователей.

Можно управлять ролями.

Но еще нельзя создавать сущности.

---

## Users & Roles

Цель:

Реализовать управление пользователями.

---

Функции:

Создание пользователя.

Редактирование пользователя.

Блокировка пользователя.

Смена пароля.

Восстановление пароля.

Управление ролями.

Журнал входов.

Настройки пользователя.

Настройки пользователя:

RTL / LTR.

Стартовая страница.

---

# Этап 3

## Pages Builder

Цель:

Создать конструктор страниц.

---

Администратор может:

Создать страницу.

Изменить страницу.

Удалить страницу.

Изменить порядок страниц.

Создать вложенное меню.

---

После создания страницы:

Она автоматически появляется в меню.

---

Результат:

Полностью динамическое меню.

Без hardcoded страниц.

---

## Entities Builder

Цель:

Создать конструктор сущностей.

---

# Финальное требование

На каждом этапе необходимо соблюдать следующие правила:

1. API First.

2. Database Driven UI.

3. Metadata Driven Architecture.

4. No Hardcoded Business Logic.

5. No Hardcoded Pages.

6. No Hardcoded Roles.

7. No Hardcoded Statuses.

8. No Translation Files.

9. All Configuration Stored In PostgreSQL.

10. Любая новая функция должна быть реализована как часть конструктора, а не как отдельный жестко зашитый модуль.
